# Data and Privacy Policy

## Principle

Use public, synthetic, or anonymized demo data whenever possible.

Do not include:

- private chat logs;
- unauthorized lab or company documents;
- real names, phone numbers, emails, student IDs, addresses;
- sensitive project materials;
- confidential research data;
- submission-anonymity-sensitive information.

## Recommended Data Sources

Preferred:

- synthetic expert materials;
- public review guidelines;
- public code-review rules;
- public project workflow templates;
- manually written demo documents.

Use with caution:

- real group discussions;
- real mentor feedback;
- internal project documents.

## Data Record

For each dataset, record:

- source;
- public or private;
- whether it contains personal information;
- anonymization status;
- demo permission;
- open-source permission.

## Output Check

Generated skill packages should be checked for:

- leaked names;
- private project details;
- unauthorized organizational information;
- sensitive behavioral descriptions.
