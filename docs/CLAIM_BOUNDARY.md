# Claim Boundary

## What We Can Claim

We can claim:

- The MVP runs end to end on the current workspace.
- Expert materials can be converted into full skill, evidence map, rule ledger, and compact skill artifacts.
- Compact skill content affects mock agent and RightCode GPT outputs.
- Harbor / verifier feedback can be converted into a unified execution report.
- `missing_rule` failures can patch domain rules into compact v2.
- `output_format_error` failures can patch the output contract.
- A minimal policy comparison has started.

## What We Should Not Claim Yet

We should not claim:

- `rule_ledger` alone is a strong new method.
- This is a general skill self-evolution framework.
- Two API cases are a benchmark.
- Current policy comparison proves large-scale superiority.
- RightCode GPT results prove general LLM stability.
- Cost optimization is fully solved.

## Safe Wording

Use:

```text
This prototype demonstrates an expert-material-first skill deployment loop where material evidence, execution feedback, and cost constraints are represented at rule level and used to generate compact skill revisions.
```

Avoid:

```text
We propose a general self-improving skill framework.
```

Avoid:

```text
Our method outperforms existing skill learning systems.
```

## Current Research Opening

The promising direction is not the ledger file itself. The promising direction is:

```text
How different decision policies use material evidence, execution evidence, and cost constraints to decide keep / drop / patch / rewrite.
```
