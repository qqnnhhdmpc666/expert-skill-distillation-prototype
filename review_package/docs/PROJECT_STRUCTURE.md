# Project Structure

## `demo/`

Streamlit display layer. It should show results, process, evidence entrypoints, and design notes. It should not be the only source of execution truth.

## `agents/`

Agent implementations and wrappers.

- `local_security_review_agent.py`: local file-IO agent that reads target assets and skill packages, then writes output, trace, stdout, and metadata.
- `api_review_*`: earlier API review mock/LLM agents.

## `backends/`

Backend boundary definitions:

- `offline_deterministic`
- `live_llm_text`
- `local_real_agent`
- `sandbox_agent`

## `data/task_cases/`

Datasetized controlled cases for the generalization suite. Each case has:

- `case.yaml`
- `source_materials/`
- `target_asset/`
- `expected_behavior.yaml`
- `verifier_contract.yaml`

## `scripts/`

Command-line runners and validators. Core nightly commands include:

- `validate_task_cases.py`
- `run_generalization_suite.py`
- `run_ablation_suite.py`
- `run_system_acceptance.py`
- `export_review_package.py`
- `validate_review_package.py`

## `runs/`

Per-run artifacts, including generated skills, attempts, verifier reports, revision decisions, and traces.

## `outputs/validation/`

Validation summaries:

- `generalization_suite.json/md`
- `ablation_summary.json/md`

## `reports/`

Human-readable status reports and overnight logs.

## `docs/`

Design docs, claim boundaries, trace visualization design, quickstart, and review guides.

## `review_package/`

External review package generated by `scripts/export_review_package.py`.
