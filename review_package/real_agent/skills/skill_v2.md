# Objective
构建并执行外部专家材料安全审查：把专家规则转为可验证检查项，审查目标资产中的文件上传、错误响应、审计与配置风险，输出带证据、修复建议和限制说明的报告。

# Inputs
- 专家材料：安全规则、约束、禁止项。
- 目标资产：代码、配置、接口描述、返回值或部署说明。
- 可选：业务上下文、允许文件类型、日志保留要求、运行环境。

# Procedure
1. 提取专家规则，转为检查项：
   - 文件上传需同时校验 MIME、内容魔数、扩展名，并使用隔离存储。
   - 用户可控文件名不得直接拼接公开目录。
   - 上传、下载、删除需记录审计日志，并设置保留周期。
   - 错误响应不得泄露内部路径、邮箱、token、api_key。
2. 对目标资产逐项匹配证据，标记：符合、不符合、证据不足。
3. 评估风险等级：可导致任意文件上传、公开访问、敏感信息泄露或审计缺失的列为高风险。
4. 为每个问题给出可执行修复：
   - 使用白名单扩展名、服务端 MIME 检测、文件头魔数校验。
   - 生成服务端随机文件名，禁止使用原始 filename 拼接路径。
   - 存储到非公开隔离目录，通过受控下载接口访问。
   - 关闭 debug，移除 debug_path，统一错误响应。
   - 配置 audit_log_retention_days 为明确正整数。
5. 输出优化前/后效果：说明原始缺陷与修复后安全属性变化。

# Evidence Protocol
- 每条发现必须引用目标资产中的具体证据，如 `filename.endswith`、`/public/uploads/`、`debug_path`、`debug=true`、`audit_log_retention_days=null`。
- 不得臆造不存在的代码行；证据不足时明确说明。
- 证据、影响、修复建议需一一对应。

# Output Contract
输出任务报告，包含：
- 审查结论与风险摘要。
- 发现列表：编号、风险等级、证据、影响、修复建议。
- 优化前后对比。
- 限制说明：未获得完整源码、运行时配置或日志策略时，结论仅基于已提供材料。

# Failure Modes
- 仅复述专家材料，未映射到目标资产。
- 无证据下结论。
- 忽略 MIME、内容魔数、扩展名、隔离存储任一上传约束。
- 忽略公开目录、debug_path、token、api_key、内部路径泄露。
- 未给出审计日志保留周期修复建议。
