# SKILL.md

## Objective
对外部专家材料与目标资产进行受控安全审查，提炼可执行规则，并产出带证据、风险等级、修复建议、残余风险与限制说明的报告；避免臆测，所有结论必须可追溯到输入文本。

## Inputs
- `expert_material`: 专家给出的安全要求、风险知识或合规约束。
- `target_asset`: 待审查代码、配置、接口描述、日志、架构说明等。
- 可选：`scope`、`risk_model`、`environment`、`assumptions`。

## Procedure
1. 提取专家材料中的强制控制点，转为审查规则：条件、判定标准、违规表现、影响。
2. 解析目标资产，标记与规则相关的证据片段。
3. 对每条规则执行匹配：
   - 满足：记录为已覆盖。
   - 违反：记录风险、证据、影响路径。
   - 信息不足：标为待确认，不作确定性结论。
4. 对文件上传类资产重点检查：
   - 是否同时校验 MIME、内容魔数、扩展名。
   - 是否使用隔离存储，避免直接暴露到公开目录。
   - 是否避免用户可控文件名直接拼接路径。
   - 上传、下载、删除是否有审计日志与保留周期。
   - 错误响应是否泄露内部路径、邮箱、token、api_key。
5. 生成修复建议：优先给最小可行修复，再给增强方案。
6. 给出限制说明：输入缺失、无法验证项、需人工复核项。

## Evidence Protocol
- 每个发现必须引用目标资产原文或等价片段。
- 不得把专家材料当作目标资产证据。
- 若仅能从描述推断，标注“基于描述”。
- 禁止编造代码行号、配置项或运行结果。
- 敏感信息展示时脱敏，仅保留必要上下文。

## Output Contract
输出 Markdown 报告，包含：
1. 摘要：总体风险、关键问题数量。
2. 规则映射表：规则、状态、证据。
3. 发现清单：标题、等级、证据、影响、修复建议。
4. 优化前后对比：原行为与建议行为。
5. 限制说明与待确认问题。

风险等级使用：Critical / High / Medium / Low / Info。

## Failure Modes
- 输入缺失：请求补充或仅输出可判定部分。
- 证据不足：标为“待确认”，不得定性为漏洞。
- 专家材料冲突：列出冲突点并采用更保守安全要求。
- 目标资产过于抽象：输出假设边界与验证清单。
- 发现潜在敏感数据：报告中脱敏，不扩散原值。
