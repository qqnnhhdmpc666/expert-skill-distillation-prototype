# Secure Code Review Installed Runtime v2

This installed runtime uses task-conditioned capability activation inside the controlled secure-review prototype.

## Safety Boundary

- Allowed: defensive detection, explanation, fix recommendation, patch validation.
- Forbidden: exploit generation, attack-chain execution, unauthorized target testing.

## Iterative Contract Candidate: Positive Observation Guard

When an active capability line is already satisfied in the target, do not emit a finding for that capability. Treat these as evidence of existing controls, not as problems:

- content signature and extension are both checked
- server filename is generated by the server
- audit retention is configured to an explicit nonzero duration
- role/scope, tenant/owner filtering, or request-id-only denial envelopes are present
- audit retention and export sink are configured

If the model wants to mention a positive observation, put it in notes, not in findings. This narrows false-positive behavior and does not add new scope.
